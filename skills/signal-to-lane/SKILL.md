# Signal-to-Lane Campaign Routing Skill

## Purpose

Use this skill to convert live platform demand signals into temporary campaign structures, production packets, performance logs, and permanent content-library routing decisions.

## Core Rule

The platform tests pressure.

The library preserves meaning.

## When To Use

Use this skill when a creator finds a live platform signal such as:

- TikTok Creator Search Insights
- YouTube search trend
- recurring comment demand
- high-performing post pattern
- platform content gap
- public debate trend

## Do Not

Do not create a permanent content lane immediately.

Do not treat platform demand as identity.

Do not chase rage.

Do not skip performance logging.

## Required Output

When creating a campaign, produce:

1. Campaign name
2. Campaign description
3. Campaign rules
4. Routing logic
5. Search insight notes
6. Content angles
7. Hook library
8. Script batch
9. Posting sequence
10. Published log template
11. Performance log template
12. Cross-lane routing files

## Default Folder Structure

- 00_campaign_manifest
- 01_search_insights
- 02_content_angles
- 03_scripts
- 04_post_queue
- 05_published_posts
- 06_performance_notes
- 07_cross_lane_promotions
- 08_screenshots

## Routing Logic

Route content after public testing.

Possible route types:

- platform-only test
- permanent lane asset
- long-form essay
- short-form series
- YouTube video
- public post
- discarded experiment
- new campaign
- new permanent lane

## Final Principle

Do not let the platform become the library.

Use the platform to discover pressure.

Use the library to preserve meaning.
